# DeepaBMS Agent Guidelines

## Environment (Termux Specific)
- **Shebangs**: Fix script shebangs with `termux-fix-shebang scripts/**/*.sh`
- **Phantom Processes**: Android 12+ may kill Expo Metro; mitigate by:
  - Keeping RAM free (`free -m`)
  - Clearing npm cache regularly (`npm cache clean --force`)
  - Disabling killer via ADB: `adb shell "settings put global settings_enable_monitor_phantom_procs false"`
- **Metro Bundler**: Use tunnel in Termux: `npx expo start --tunnel`

## Project Structure
- **Main App**: Expo React Native (root `App.tsx`, source in `src/`)
- **Backend**: Node.js/Express (`apps/backend/`)
- **Windows App**: Electron (`apps/windows/`)

## Key Commands
- **Dev Server**: `npm start` (Expo) or `npx expo start --tunnel` (Termux tunnel)
- **Android Build**: Use EAS (local builds not supported in Termux):
  - Preview: `eas build --platform android --profile preview --non-interactive`
  - Production: `eas build --platform android --profile production --non-interactive`
  - Login: `eas login --no-browser`
- **Web**: `npm run web` (`expo start --web`)
- **TypeScript**: `npm run ts:check` (`tsc`)
- **Lint**: `npm run lint` (ESLint), `npm run lint:fix` (auto-fix)
- **Format**: `npm run format` (Prettier)
- **Test**: `npm test` (Jest)
- **Backend Dev**: `cd apps/backend && npm run dev`
- **Backend Start**: `cd apps/backend && npm start`
- **Windows App**: `cd apps/windows && npm start`

## Testing
- **Framework**: Jest with `jest-expo` preset
- **Test Files**: Place in `tests/unit/` (logic) or `tests/components/` (components), or use `.test.tsx`/`.spec.tsx` suffix
- **Run**: `npm test`

## Code Quality
- **Linting**: ESLint with `eslint-config-expo` and Prettier plugin
  - Ignores: `node_modules/`, `dist/`, `android/`, `ios/`, `.expo/`
- **Formatting**: Prettier (config in `.prettierrc`)
  - `singleQuote: true`, `trailingComma: "all"`, `semi: true`, `tabWidth: 2`, `printWidth: 80`
- **TypeScript**: Strict mode via `tsconfig.json` (extends `expo/tsconfig.base`)

## Scripting Guidelines (scripts/ directory)
- Shebang: `#!/usr/bin/env bash` (adjusted via termux-fix-shebang)
- Strict mode: `set -euo pipefail`
- Path detection:
  ```bash
  SCRIPT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
  PROJECT_DIR="$(cd "$SCRIPT_DIR/../.." && pwd)"
  ```
- Source utils: `source "$PROJECT_DIR/scripts/lib/utils.sh"`
- Cleanup trap: `trap cleanup EXIT INT TERM`

## Backend Specifics
- Build: `npm run build` (transpiles TS to JS in `dist/`)
- Start: `node dist/index.js`
- Environment: Uses `.env` (via dotenv)

## Windows App Specifics
- Packaging: `npm run package` (uses electron-builder for NSIS target)
- Icon: `apps/windows/assets/icon.ico`

## Critical Implementation Notes

### State Management
- **Single Large Reducer**: The `StoreContext.tsx` reducer exceeds 1100 lines and handles all app state. Consider splitting into feature-specific reducers or using Zustand/Jotai for better maintainability.
- **Performance**: The initial dataset generated by `buildSeed()` is substantial and may impact startup time on lower-end devices.

### UI/UX Concerns
- **DayBook.tsx Line 459**: Contains a misplaced closing `</Card>` tag inside a TouchableOpacity that may cause rendering issues.
- **Missing Error Boundaries**: No error boundaries detected in the component tree - runtime errors could crash the entire app.
- **Accessibility**: Some touchable components lack accessibility labels, impacting screen reader usability.

### Data Synchronization
- **Backend Dependency**: The sync function in StoreContext.tsx references `/api/sync` endpoint. Verify this endpoint exists in the backend implementation.
- **Sync Timing**: Uses `setTimeout` for status reset which could create race conditions with rapid successive sync attempts.

### Security Considerations
- **PIN Validation**: Login PIN comparison uses simple string equality (vulnerable to timing attacks). For higher security implementations, consider constant-time comparison.

### Platform-Specific Implementations
- **Thermal Printing**: Uses `expo-print` which works differently on web vs native. Verify print functionality across target platforms.
- **Date Handling**: Numerous date utility functions exist - ensure consistent usage throughout the codebase.

### Development Best Practices
1. **Component Size**: Monitor component file sizes - several screens exceed 300-400 lines and may benefit from decomposition.
2. **Dependency Arrays**: Verify all `useEffect`, `useCallback`, and `useMemo` hooks have correct dependency arrays.
3. **Error Handling**: Async operations (especially network requests) should have proper error handling and user feedback.
4. **Testing Coverage**: Critical paths like authentication, financial transactions, and sync operations should have comprehensive test coverage.
5. **Accessibility**: Add `accessibilityLabel` / `accessibilityRole` to all touchable elements; an `ErrorBoundary` component is now in `src/components/ErrorBoundary.tsx` and wraps the app root.

3. Error boundaries and error handling
4. Accessibility audits
5. Performance optimization of large lists
